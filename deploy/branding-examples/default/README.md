# Default Pyrite branding

This folder documents the white-label branding config format. It is
**not** loaded automatically — Pyrite's defaults are compiled into the
`BrandingService`. Point `PYRITE_BRANDING_DIR` at a copy of this folder
(or an adaptation of it) to override them.

See `docs/deployment/white-labeling.md` for the full walkthrough.

## Sibling example

`../transparency-cascade/` is a production-ready branding folder for
the Transparency Cascade Press instance — use it as a reference for
shipping a fully-branded deployment with logo + wordmark assets.
